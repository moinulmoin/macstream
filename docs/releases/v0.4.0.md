# MacStream v0.4.0

MacStream v0.4.0 adds multi-destination streaming while keeping capture and
composition shared. One program output can now publish to as many as three
RTMP or RTMPS destinations without allowing one slow target to stall the
others.

## Highlights

- Configure, enable, disable, and persist up to three streaming destinations.
- Publish one composed screen and webcam output to selected destinations.
- Track connection, throughput, queue pressure, failure, and reconnect state
  independently for every destination.
- Keep each publisher on a bounded append lane with visible degraded state
  when a destination falls behind.
- Continue streaming to healthy destinations when another target fails to
  connect or disconnects.
- Store stream keys per destination in Keychain and keep endpoint secrets out
  of logs, events, UI status, and exported reports.

## Reliability and Validation

- Initial destination connections run concurrently and tolerate partial
  failure when at least one destination is publishing.
- Runtime reconnects are independent and bounded per destination.
- Stream startup cancellation and publisher shutdown are idempotent under
  concurrent lifecycle changes.
- The release matrix builds default, HaishinKit, MLX, and combined variants,
  runs the full combined test suite, exercises real RTMP publishing against a
  local ingest, and validates the packaged application bundle.

## Current Limits

MacStream remains a personal-use prototype. Validate important broadcasts with
a private or test destination first. Presenter cutout composition, video
editing, transcription, and other AI-assisted features are not included in
this release.
