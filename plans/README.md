## Direction rationale

The AI sequence is deliberate:

1. Add OpenAI-compatible local provider first: it unlocks LM Studio, Ollama, llama.cpp, MLX server, and similar local runtimes without MacStream owning model downloads, GPU lifecycle, crash isolation, or hot-path contention.
2. Ship director explanations and deterministic preflight coach next: visible AI value, no model-controlled live switching, and no live encoder dependency.
3. Defer Foundation Models until the provider settings/status/fallback contract is proven.
4. Defer VLM/sampled-frame work until the core publish/recording path is more proven; when it happens, it must be a slow typed-signal adapter, not an AI director.
