# MacStream v0.6.0

MacStream v0.6.0 tightens the path from first launch to a recoverable live
session and removes demonstrated work from the media path without changing
encoded output quality.

## Workflow

- Ask macOS directly for Camera and Microphone access when permission has not
  been decided.
- Route denied access to the matching System Settings pane and explain that
  Screen Recording requires reopening MacStream.
- Keep RTMP as the selected intent when setup is incomplete and open the
  Destination settings directly.
- Validate every enabled destination, not only the destination currently
  selected in Settings.
- Show concrete actions for failed starts, reconnecting streams, exhausted
  recovery, and current destination backpressure.

## Native Camera Integration

- Discover built-in, external, Continuity Camera, and Desk View devices through
  one shared capture path.
- Report public macOS status for Continuity Camera, Center Stage, Portrait,
  Studio Light, Background Replacement, and Reactions.
- Open Apple's Video Effects interface for system-owned controls.

MacStream does not use private APIs or present Portrait, Studio Light,
Background Replacement, and Reactions as app-controlled toggles.

## Performance And Reliability

- Measure the microphone level at a bounded visual cadence while preserving all
  audio delivery and freshness tracking.
- Read current RTMP queue state before composing each shared output frame.
- Skip composition only when every active publisher is saturated and no local
  recording needs the frame.
- Resume rendering immediately when any destination drains.
- Keep local recording success separate from first-frame RTMP delivery.
- Refresh aggregate queue health immediately as lanes degrade or recover.

## Validation

- 401 default tests pass.
- 402 HaishinKit release-path tests pass, including real-publisher integration
  coverage compiled into the variant.
- Default, HaishinKit, MLX, and combined HaishinKit plus MLX builds pass.
- Packaged BRB-idle sampling reported 0.0% CPU, 139.7-140.7 MB resident
  memory, and 7 threads across 16 samples in 30 seconds. Live device and RTMP
  measurements remain part of operator smoke testing.
- Publication is gated on Developer ID signing, hardened runtime, notarization,
  stapling, Gatekeeper validation, checksums, and Sparkle update verification.

## Current Limits

MacStream remains a personal-use prototype. Validate important broadcasts with
a private or test destination first. Video editing, transcription, AI setup,
and model-controlled live behavior remain deferred.
