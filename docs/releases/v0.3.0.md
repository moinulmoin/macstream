# MacStream v0.3.0

MacStream v0.3.0 establishes the first signed, notarized, streaming-first
release. It concentrates on the single-destination live path, composited output,
performance controls, and the diagnostics needed to validate longer sessions.

## Highlights

- Hardened RTMP/RTMPS publishing with bounded runtime reconnect behavior.
- Shared Screen + Webcam composition across preview, streaming, and recording.
- Optional record-while-streaming with explicit recording ownership.
- A/V drift, dropped-frame, RTMP queue, throughput, and recovery observability.
- Preview quality controls independent from encoded stream quality.
- Signed, notarized, and stapled DMG for first-time installation.
- Sparkle-signed ZIP for in-app updates.

## Reliability and Validation

- The release CI exercises the real HaishinKit publisher against a local RTMP
  ingest and validates decoded H.264 output.
- Media append queues are bounded and report backpressure instead of growing
  without limit.
- The release workflow verifies Developer ID signing, hardened runtime,
  notarization, stapling, Gatekeeper acceptance, and SHA-256 checksums.

## Current Limits

MacStream remains a personal-use prototype. Validate important broadcasts with
a private or test destination first. Multi-destination streaming, presenter
cutout, native camera-effect controls, video editing, and AI features are not
included in this release.
