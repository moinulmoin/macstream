# v0.6 Workflow Polish And Measured Performance

MacStream v0.6 improves the path from first launch to a recoverable live
session. It also removes demonstrated media-path work without weakening output
fidelity, capture freshness, or bounded RTMP delivery.

## Product Outcomes

1. A first-time camera or microphone permission can be requested directly.
   Denied permissions route to the matching System Settings pane, and Screen
   Recording continues to explain its relaunch requirement.
2. Selecting RTMP remains an explicit user intent. An incomplete destination
   guides the user to configure RTMP instead of silently returning to Preview.
3. Failed, reconnecting, and backpressured streams show a concrete next action
   derived from existing runtime state. Stream keys and endpoint secrets remain
   redacted.
4. MacStream reports the public native camera effects available or active on
   the selected camera and can open Apple's Video Effects interface. It does
   not present system-owned effects as app-controlled toggles.
5. Microphone delivery remains lossless while level-meter calculation is
   bounded to a human-visible cadence.
6. A fully saturated RTMP fan-out does not spend CPU rendering a frame that no
   publisher can accept. Any available publisher or active local recorder keeps
   the shared render path enabled.

## Public Camera API Boundary

The release may use public AVFoundation capability and status APIs for Center
Stage, Portrait, Studio Light, Background Replacement, reactions, Continuity
Camera, and the system Video Effects interface.

The release must not use private Control Center APIs or claim direct control of
Portrait, Studio Light, Background Replacement, system gesture reactions,
manual framing, hardware zoom, or Smart Framing where macOS exposes no public
setter.

## Performance Evidence

- Behavior tests must prove microphone freshness and media delivery are not
  throttled with the level meter.
- Behavior tests must prove all-lane saturation skips unnecessary composition,
  while one accepting lane or an active recorder preserves composition.
- The full default and HaishinKit test suites must pass.
- Release builds must pass in default, HaishinKit, MLX, and combined feature
  configurations.
- Short same-machine packaged measurements should record CPU, resident memory,
  thread count, capture FPS, dropped frames, queue pressure, and A/V drift.
  Hardware-specific percentages remain evidence, not CI thresholds.

The release-candidate BRB-idle package recorded CPU, resident memory, and
thread count in `benchmarks/v0.6.0-packaged-idle.csv`. Capture FPS, dropped
frames, queue pressure, and A/V drift require the operator's granted devices
and RTMP destination and remain part of the post-install smoke pass.

## Out Of Scope

- Video editing or post-production.
- AI setup, transcription, summaries, or model-controlled live behavior.
- Private camera-effect APIs or simulated native-effect controls.
- A raw screen-publish bypass that could diverge from preview or recording
  geometry.
- Speculative refactors without a measured workflow or runtime benefit.

## Release Gate

v0.6.0 ships only after implementation, focused behavior tests, the complete
build/test matrix, packaged UI inspection, adversarial review, release notes,
Developer ID signing, Apple notarization and stapling, public artifact checksum
validation, Gatekeeper validation, and Sparkle appcast verification.
